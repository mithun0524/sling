# sling v1.1 — edit, prompts, clipboard, env secrets

**Date:** 2026-07-15
**Status:** Approved, implementing

Builds on v1 (`add`/`run`/`ls`/`show`/`rm`). Four additions, all extending existing modules.

## 1. `sling edit <name>`

Open the stored request as pretty JSON in `$EDITOR` (fallback `vi`). On save:
- Parse the edited text as JSON. Broken JSON → error, keep original untouched.
- Re-run `detectVars` on the edited request so var list stays accurate.
- Persist.

## 2. Prompt for missing vars on `run`

After substitution, if normal (`{{name}}`) vars are still unfilled:
- **Interactive TTY:** prompt once per missing var (`id? › `), collect answers, re-substitute, then fire.
- **Non-TTY (piped/scripted):** keep the v1 hard-fail (`missing vars: …`), fire nothing. Preserves scriptability.

## 3. Clipboard read on `add`

`sling add <name>` with no piped stdin reads the system clipboard (`pbpaste` on macOS).
If stdin IS piped, use it (v1 behavior). Empty clipboard + no pipe → error.

## 4. Env-var secrets: `{{env:NAME}}`

New template form `{{env:NAME}}` (NAME matches `[A-Z_][A-Z0-9_]*`).
- Resolved from `process.env` at **run** time. Never prompted, never `--var`-filled.
- Missing env var → reported as missing: `env:NAME (set NAME in your environment)`. Fire nothing.
- `detectVars` does NOT list `env:` forms as fillable vars (they're auto).
- Lets you store `authorization: Bearer {{env:API_TOKEN}}` — the secret stays out of
  `~/.config/sling/requests.json`.

## Module changes

| Module | Change |
|---|---|
| `templater` | `substitute(req, vars, env)` also resolves `{{env:NAME}}` from `env`; missing env labeled `env:NAME`. `detectVars` unchanged (ignores env forms). |
| `commands` | `IO` gains `readClipboard()`, `prompt(q)`, `isTTY`, `openEditor(text)`. `cmdAdd` falls back to clipboard; `cmdRun` prompts on TTY + passes `process.env`; new `cmdEdit`. |
| `cli` | Wire real clipboard (`pbpaste`), readline prompt, `$EDITOR` spawn, `process.stdin.isTTY`; route `edit`. |

## Error handling (additions)

| Case | Behavior |
|---|---|
| `edit` with broken JSON | Error, original request unchanged. |
| `edit` unknown name | Error, suggest `sling ls`. |
| `add` no pipe + empty clipboard | Error: nothing to read. |
| `run` missing `{{env:X}}` | Error listing `env:X`; fire nothing (even on TTY — env is never prompted). |
| `run` missing normal var, non-TTY | v1 hard-fail. |

## Testing

- `templater`: `{{env:X}}` resolves from passed env; missing env → `missing: ["env:X"]`; normal + env mixed.
- `commands`: `cmdAdd` uses clipboard when stdin empty; `cmdRun` prompts missing vars via injected `prompt` on TTY, hard-fails when `isTTY=false`; env missing fires nothing; `cmdEdit` validates JSON, re-detects vars, rejects broken. All via fake IO — no real editor/clipboard/network.
